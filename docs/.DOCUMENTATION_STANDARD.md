## 📋 Template de Header

Todos os documentos devem seguir este template:

```markdown
# Título do Documento

**Data:** YYYY-MM-DD  
**Status:** Ativo | Histórico | Obsoleto  
**Versão:** X.Y (se aplicável)  
**Categoria:** Guia | Arquitetura | API | Política | Relatório  
**Audiência:** Desenvolvedores | DevOps | Stakeholders | Todos

---
```

---

## 🏷️ Categorias

- **Guia:** Tutoriais e how-tos práticos
- **Arquitetura:** Decisões e padrões arquiteturais
- **API:** Documentação de endpoints e integrações
- **Política:** Regras, políticas e compliance
- **Relatório:** Análises, auditorias e validações
- **Histórico:** Documentos arquivados (em `archive/`)

---

## 👥 Audiências

- **Desenvolvedores:** Documentação técnica para desenvolvimento
- **DevOps:** Guias de infraestrutura, deploy e CI/CD
- **Stakeholders:** Visão estratégica e governança
- **Todos:** Documentação geral acessível a todos

---

## 📝 Regras de Formatação

### Títulos

- Use `#` para H1 (título principal)
- Use `##` para H2 (seções principais)
- Use `###` para H3 (subseções)
- Sempre adicione linha em branco após títulos

### Listas

- Use `-` para listas não ordenadas
- Use `1.`, `2.`, etc. para listas ordenadas
- **2 espaços** após marcadores (padrão MD030)

### Código

- Use \`backticks\` para código inline
- Use blocos \`\`\` com linguagem especificada
- Sempre especifique linguagem em blocos de código

### Links

- Use links relativos para documentos internos: `[texto](./arquivo.md)`
- Use links absolutos para recursos externos
- Valide links regularmente

## 📂 Estrutura de Pastas

```
docs/
├── README.md                    # Índice principal
├── archive/                     # Documentos históricos
│   └── README.md               # Índice de arquivados
├── adr/                        # Architecture Decision Records
│   └── 000X-*.md
└── [documentos ativos].md      # Documentação principal
```

## ✅ Checklist de Qualidade

Antes de commitar um documento:

- [ ] Header padronizado presente
- [ ] Data e status atualizados
- [ ] Categoria e audiência definidas
- [ ] Links internos validados
- [ ] Formatação markdown correta
- [ ] Sem erros de lint (markdownlint)
- [ ] Referenciado no `docs/README.md` (se relevante)

---

## 🔄 Processo de Arquivamento

Documentos devem ser arquivados quando:

1. **Processo concluído:** Migrações, validações, auditorias
2. **Substituído:** Documento antigo substituído por versão nova
3. **Obsoleto:** Informação não mais relevante

**Ação:** Mover para `docs/archive/` e atualizar `docs/README.md`
